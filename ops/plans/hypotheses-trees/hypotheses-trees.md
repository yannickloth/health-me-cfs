# Hypothesis Trees — ME/CFS Paper

Root registry of all hypotheses generated by `/integrate-topic` brainstorm phases, organized as trees by originating topic. Updated automatically after every Phase 3 brainstorm and Phase 4 recursive integration.

---

## How to read this file

Each topic that has run through `/integrate-topic` has a subtree file in `subtrees/`. This file lists those subtrees. Each node in a subtree is one brainstorm idea, carrying:

- **ID** — brainstorm ID (e.g. `1.1`, `2.3`)
- **Title** — short name
- **Env** — target environment (`hypothesis` / `speculation` / `open-question` / `proposal`)
- **Cert** — certainty 0.0–1.0 (`n/a` for proposals)
- **Usefulness** — scores 0–5 per dimension (see legend below)
- **Status** — integration state
- **Children** — IDs of hypotheses spawned by this node's own `/integrate-topic` run

### Usefulness dimension legend

| Code   | Dimension             | What 5 means                                               |
|--------|-----------------------|------------------------------------------------------------|
| `mech` | Mechanistic insight   | Core causal pathway; directly advances pathophysiology model |
| `tx`   | Treatment potential   | High-confidence actionable therapeutic target              |
| `expl` | Explanatory power     | Explains multiple unexplained ME/CFS features at once      |
| `math` | Math model fit        | Adds a new variable, equation, or parameter to ODE/DAG     |
| `dx`   | Diagnostic/biomarker  | Validated or near-validated clinical measurement           |

### Status values

| Symbol | Meaning                                       |
|--------|-----------------------------------------------|
| `⬜`   | pending — not yet started                     |
| `🔵`   | in progress                                   |
| `✅`   | integrated into paper                         |
| `⏭️`   | queued — waiting for current cycle to finish  |
| `🚫`   | skipped — no ME/CFS relevance                 |
| `↩️`   | parked — insufficient literature, open-question only |
